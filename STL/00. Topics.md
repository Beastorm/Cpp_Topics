## 2. Containers ğŸ§±

Containers store collections of objects.

### 2.1 Sequence Containers

Store elements in **order of insertion**.

- **`vector`**
  - Dynamic array, contiguous memory.
  - Key functions: `push_back`, `pop_back`, `size`, `capacity`, `reserve`, `emplace_back`.

- **`deque`**
  - Double-ended queue, fast insertion/removal at **both ends**.

- **`list`**
  - Doubly-linked list, fast insertion/removal anywhere (but not random access).

- **`forward_list`** (C++11)  
  - Singly-linked list.

- **`array`** (C++11)  
  - Fixed-size array (stack-allocated, `std::array<T, N>`).

- **`string`**
  - Sequence of characters with STL-like methods (`push_back`, `substr`, etc.).

---

- Containers:
  - `map` / `multimap` â†’ keyâ€“value pairs  
  - `set` / `multiset` â†’ unique / non-unique keys

- **Key operations**: `insert`, `find`, `erase`, `count`

### 2.3 Unordered Containers (C++11) ğŸ”‘

- Examples:  
  - `unordered_set`

  - `unordered_multiset`, `unordered_multimap`

- **Average complexity:**
  - `insert` â†’ `O(1)`

  - `erase`  â†’ `O(1)`

- `queue` â†’ **FIFO** (usually uses `deque`)  
- `priority_queue` â†’ **max-heap** (or min-heap with custom comparator)

## 3. Iterators ğŸ¯

Iterators provide a uniform way to **traverse containers**.

### Types of Iterators

- **Input Iterator** â€“ read-only, one-pass  
- **Output Iterator** â€“ write-only, one-pass  
- **Forward Iterator** â€“ read/write, multi-pass  
- **Bidirectional Iterator** â€“ move forward and backward (`list`, `set`, `map`)  
- **Random Access Iterator** â€“ jump by index (`vector`, `deque`, `array`)  

### Common Operations

- `++it`, `--it`  
- `*it` (dereference)  
- `it + n`, `it - n` (for random access iterators)

---

## 4. Algorithms âš™ï¸

Predefined **generic operations** on containers (work with iterators).

### Categories

- **Non-modifying algorithms:**  
  `for_each`, `count`, `find`, `binary_search`

- **Modifying algorithms:**  
  `copy`, `replace`, `fill`, `swap`, `remove`, `unique`

- **Sorting / Partitioning:**  
  `sort`, `partial_sort`, `stable_sort`, `nth_element`, `partition`

- **Set operations (on sorted ranges):**  
  `set_union`, `set_intersection`, `set_difference`

- **Numeric algorithms (in `<numeric>`):**  
  `accumulate`, `inner_product`, `adjacent_difference`, `iota`

---

## 5. Allocators ğŸ§ 

Allocators control **how memory is allocated** for containers.

- Default: `std::allocator<T>`
- Custom allocators can be provided for specific memory strategies.
- Rarely used directly in most application code.

---

## 6. Iterators + Algorithms Patterns ğŸ”„

STL emphasizes separating:

- **Data structure (Container)**  
  from  
- **Operation (Algorithm)**

using **iterators**.

Examples:

- `std::copy(begin, end, dest_begin)`
- `std::transform(begin, end, out_begin, func)`
- `std::accumulate(begin, end, init)`

This supports **generic programming**:  
same algorithms work with different containers as long as they provide the right iterator category.

## 7. Advanced STL Topics ğŸŒŸ

- **Move semantics**
  - `std::move`, `emplace_back`, `emplace` families

- **Smart pointers (in `<memory>`)**
  - `std::unique_ptr`
  - `std::shared_ptr`
  - `std::weak_ptr`

- **STL Performance Patterns**
  - Prefer `vector` over `list` in most cases (cache-friendly, less allocation)
  - Be mindful of iterator invalidation rules

- **Custom comparators**
  - For `set`, `map`, `priority_queue` etc. using functors or lambdas

- **Ranges and Views (C++20)**
  - `std::ranges::filter_view`, `transform_view`, `views::iota`, etc.
