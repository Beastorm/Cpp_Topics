## 🟢 Stack (Automatic Storage)

The fastest, safest, and most common way. Object lives in the current scope — destructor runs automatically when the scope exits.

### 1. Direct Initialization

```cpp
Dog d("Rex");          // calls Dog(const char*)
Dog d{"Rex"};          // brace form — same result, prevents narrowing
Dog d("Rex", 3);       // multiple args
```

### 2. Value Initialization

```cpp
Dog d{};               // zero-initializes POD members, calls default ctor
Dog d = Dog();         // same — explicit temporary then copy/move-elided
int x{};               // x == 0 (guaranteed)
```

### 3. Copy Initialization

```cpp
Dog d = other;         // copy constructor (or move if other is rvalue)
Dog d = Dog("Rex");    // move-constructed from temporary (NRVO/copy elision)
```

### 4. Move Initialization

```cpp
Dog d = std::move(other);    // move constructor — steals other's resources
                              // other is left in valid-but-unspecified state
```

### 5. Uniform / Brace Initialization (C++11)

```cpp
Dog d{"Rex", 3};       // works for any constructor
int arr[]{1, 2, 3};    // array on stack
std::pair<int,int> p{1, 2};
```

> **Prefer brace `{}` for stack objects** — it prevents narrowing conversions and is consistent across all types.

## 🟡 Heap (Dynamic Storage)

Object lives until explicitly freed or smart pointer goes out of scope.

### 6. Raw `new` / `new[]`

```cpp
Dog* d  = new Dog("Rex");       // single object
Dog* ds = new Dog[5];           // array of 5 default-constructed Dogs

delete d;                        // must call manually
delete[] ds;                     // array form — must match new[]
```

> ❌ Avoid raw `new` in modern C++. Use smart pointers instead.

### 7. `std::make_unique` (C++14) — preferred for sole ownership

```cpp
auto d = std::make_unique<Dog>("Rex");   // sole owner
// d is freed automatically when it goes out of scope

auto ds = std::make_unique<Dog[]>(5);    // unique_ptr to array
```

Why `make_unique` over `new`:
- Exception-safe — no leak if constructor throws
- Single allocation expression — no raw pointer exposed
- Communicates intent: "I own this exclusively"

### 8. `std::make_shared` — for shared ownership

```cpp
auto d  = std::make_shared<Dog>("Rex");  // ref count = 1
auto d2 = d;                             // ref count = 2
// freed when BOTH d and d2 go out of scope

// shared_ptr from unique_ptr (transfers ownership)
auto u = std::make_unique<Dog>("Rex");
std::shared_ptr<Dog> s = std::move(u);
```

> Use `make_shared` over `new` + `shared_ptr<T>(ptr)` — it does one allocation instead of two (object + control block combined).

### 9. `std::make_shared` with Custom Deleter

```cpp
// Custom cleanup (e.g. C library resource)
auto d = std::shared_ptr<Dog>(
    new Dog("Rex"),
    [](Dog* p) { p->shutdown(); delete p; }
);

// unique_ptr with custom deleter
auto f = std::unique_ptr<FILE, decltype(&fclose)>(
    fopen("data.txt", "r"), fclose
);
```

## 🔴 Special / In-Place Creation

### 10. Placement `new`

Constructs an object into a **pre-allocated buffer** — no heap allocation.

```cpp
alignas(Dog) std::byte buf[sizeof(Dog)];   // raw buffer on stack

Dog* d = new(buf) Dog("Rex");              // construct INTO buf — no allocation
d->bark();

d->~Dog();   // must call destructor manually — no delete!
```

Used in: memory pools, custom allocators, `std::optional` / `std::variant` internals, lock-free data structures.

```cpp
// In a memory pool
class Pool {
    alignas(std::max_align_t) char storage_[4096];
    std::size_t offset_ = 0;
public:
    template<typename T, typename... Args>
    T* create(Args&&... args) {
        void* ptr = storage_ + offset_;
        offset_ += sizeof(T);
        return new(ptr) T(std::forward<Args>(args)...);   // placement new
    }
};
```

### 11. `std::construct_at` (C++20)

The modern, `constexpr`-friendly alternative to placement `new`:

```cpp
#include <memory>

alignas(Dog) std::byte buf[sizeof(Dog)];
Dog* d = std::construct_at(reinterpret_cast<Dog*>(buf), "Rex");

std::destroy_at(d);   // paired destructor call
```

Works in `constexpr` contexts where placement `new` doesn't.

### 12. `emplace_back` / `emplace` — in-container construction

Constructs **directly inside the container's storage** — no temporary, no move:

```cpp
std::vector<Dog> dogs;
dogs.emplace_back("Rex", 3);        // Dog constructed in-place inside vector
dogs.emplace(dogs.begin(), "Fido"); // in-place at position

std::map<int, Dog> m;
m.emplace(1, "Rex");                // pair constructed in-place

// push_back vs emplace_back
dogs.push_back(Dog("Rex"));         // constructs Dog, then moves into vector
dogs.emplace_back("Rex");           // constructs Dog directly inside vector ← faster
```

### 13. Aggregate Initialization

For simple structs with no user-declared constructors:

```cpp
struct Point { int x, y; };
Point p{1, 2};            // x=1, y=2 — no constructor needed

// C++20 designated initializers — by name, any order
Point p{.x = 1, .y = 2};

struct Config {
    std::string host = "localhost";
    int         port = 8080;
    bool        tls  = false;
};
Config c{.host = "example.com", .tls = true};   // port stays 8080
```

### 14. `std::optional` — object that may or may not exist

```cpp
#include <optional>

std::optional<Dog> findDog(int id) {
    if (id == 42) return Dog("Rex");   // constructed inside optional
    return std::nullopt;               // no object
}

auto d = findDog(42);
if (d) d->bark();                      // safe access

// Construct in-place
std::optional<Dog> opt;
opt.emplace("Rex", 3);                 // in-place construction, no temp
```

## 🔵 Factory / Indirect Creation

The caller doesn't use `new` or constructors directly — a function or method handles construction.

### 15. Factory Function

A free function that constructs and returns an object:

```cpp
// Returns by value — move/NRVO means zero copies
Dog makePoliceDog(const std::string& name) {
    Dog d(name);
    d.train(TrainingProgram::Police);
    return d;   // NRVO — constructed directly in caller's memory
}

auto d = makePoliceDog("Rex");

// Factory returning smart pointer (when heap needed)
std::unique_ptr<Shape> makeShape(const std::string& type) {
    if (type == "circle")    return std::make_unique<Circle>();
    if (type == "rectangle") return std::make_unique<Rectangle>();
    throw std::invalid_argument("Unknown shape: " + type);
}
```

### 16. Named Constructor Idiom (Static Factory Method)

Constructor is private; public static methods provide named creation paths:

```cpp
class Angle {
public:
    static Angle fromDegrees(double d) { return Angle(d * M_PI / 180.0); }
    static Angle fromRadians(double r) { return Angle(r); }

    double radians() const { return rad_; }

private:
    explicit Angle(double r) : rad_(r) {}   // private ctor
    double rad_;
};

auto a = Angle::fromDegrees(90.0);    // clear intent
auto b = Angle::fromRadians(M_PI);    // clear intent
// Angle a(1.57);  ← not allowed — ambiguous: degrees or radians?
```

### 17. Virtual Constructor — `clone()` and `create()`

Since constructors cannot be virtual, use virtual methods to copy or default-construct through a base pointer:

```cpp
struct Shape {
    virtual Shape* clone()  const = 0;   // copy this object
    virtual Shape* create() const = 0;   // make blank object of same type
    virtual ~Shape() = default;
};

struct Circle : Shape {
    double radius;
    Circle* clone()  const override { return new Circle(*this); }
    Circle* create() const override { return new Circle(); }
};

// Deep-copy a heterogeneous container — no slicing
std::vector<Shape*> copyAll(const std::vector<Shape*>& src) {
    std::vector<Shape*> result;
    for (auto* s : src) result.push_back(s->clone());
    return result;
}
```

### 18. Abstract Factory / Factory Method Pattern

```cpp
struct AnimalFactory {
    virtual std::unique_ptr<Animal> createDog() = 0;
    virtual std::unique_ptr<Animal> createCat() = 0;
    virtual ~AnimalFactory() = default;
};

struct FarmAnimalFactory : AnimalFactory {
    std::unique_ptr<Animal> createDog() override {
        return std::make_unique<FarmDog>();
    }
    std::unique_ptr<Animal> createCat() override {
        return std::make_unique<BarnCat>();
    }
};

// Caller only knows AnimalFactory — concrete types are hidden
void setup(AnimalFactory& factory) {
    auto dog = factory.createDog();
    auto cat = factory.createCat();
}
```

### 19. Prototype Registry

Map type names to prototype instances. Create new objects by cloning the prototype at runtime:

```cpp
class ShapeRegistry {
public:
    void registerShape(const std::string& name, std::unique_ptr<Shape> proto) {
        registry_[name] = std::move(proto);
    }

    std::unique_ptr<Shape> create(const std::string& name) const {
        return std::unique_ptr<Shape>(registry_.at(name)->clone());
    }

private:
    std::unordered_map<std::string, std::unique_ptr<Shape>> registry_;
};

ShapeRegistry reg;
reg.registerShape("circle",    std::make_unique<Circle>());
reg.registerShape("rectangle", std::make_unique<Rectangle>());

// Create any registered type by name — no switch, no if-else
auto shape = reg.create("circle");
```

## ⚪ Less Common but Important

### 20. Static Local Variable (Meyers Singleton)

Constructed exactly once, on first call, thread-safe since C++11:

```cpp
Config& getConfig() {
    static Config instance{"default.cfg"};   // constructed once, lives forever
    return instance;
}

// Meyers Singleton pattern
class Database {
public:
    static Database& instance() {
        static Database db;    // thread-safe, lazy construction
        return db;
    }
private:
    Database() { connect(); }
};
```

### 21. `std::allocator` + `std::construct_at`

Low-level, used inside container implementations:

```cpp
std::allocator<Dog> alloc;

Dog* ptr = alloc.allocate(1);                    // raw memory — no construction
std::construct_at(ptr, "Rex");                    // construct in-place (C++20)
// or: ::new(static_cast<void*>(ptr)) Dog("Rex"); // pre-C++20

std::destroy_at(ptr);                             // call destructor
alloc.deallocate(ptr, 1);                         // free memory
```

### 22. `std::variant` — type-safe union

```cpp
#include <variant>

std::variant<int, double, std::string> v;

v = 42;                          // holds int
v = 3.14;                        // now holds double
v = std::string("hello");        // now holds string

// Construct in-place with std::in_place_type
std::variant<int, std::string> v2{std::in_place_type<std::string>, "hello"};

// Visit
std::visit([](auto& val) { std::cout << val << "\n"; }, v);
```

### 23. `std::any`

Type-erased single object — stores anything copyable:

```cpp
#include <any>

std::any a = 42;
std::any b = std::string("hello");
std::any c = Dog("Rex");

int x = std::any_cast<int>(a);                    // extract value
Dog& d = std::any_cast<Dog&>(c);                  // extract by reference

// In-place construction
std::any e{std::in_place_type<Dog>, "Rex"};
```

### 24. Lazy Initialization

Object created only when first accessed:

```cpp
// Manual lazy init
class Service {
public:
    Dog& dog() {
        if (!dog_) dog_ = std::make_unique<Dog>("Rex");   // create on demand
        return *dog_;
    }
private:
    std::unique_ptr<Dog> dog_;   // null until first access
};

// With std::optional
class Service {
public:
    Dog& dog() {
        if (!dog_) dog_.emplace("Rex");   // in-place construct inside optional
        return *dog_;
    }
private:
    std::optional<Dog> dog_;
};
```

### 25. Object Pool

Pre-allocate objects, reuse them — avoids repeated heap allocations:

```cpp
template<typename T, std::size_t N>
class ObjectPool {
public:
    template<typename... Args>
    T* acquire(Args&&... args) {
        if (free_.empty()) throw std::bad_alloc();
        T* obj = free_.back();
        free_.pop_back();
        return new(obj) T(std::forward<Args>(args)...);   // placement new
    }

    void release(T* obj) {
        obj->~T();             // explicit destructor
        free_.push_back(obj);  // return to pool
    }

private:
    alignas(T) std::byte storage_[sizeof(T) * N];
    std::vector<T*> free_;
    // constructor fills free_ with pointers into storage_
};
```

### 26. Deserialization / Reconstruction

Creating objects from external data:

```cpp
// From JSON (conceptual — using a library like nlohmann/json)
Dog fromJson(const json& j) {
    return Dog(j["name"].get<std::string>(),
               j["age"].get<int>());
}

// From binary stream
Dog fromStream(std::istream& is) {
    std::string name;
    int age;
    std::getline(is, name);
    is >> age;
    return Dog(name, age);
}
```

## Quick Syntax Cheat Sheet

| Method | Syntax | Storage | Use When |
|--------|--------|---------|----------|
| Direct init | `Dog d("Rex")` | Stack | Default — always prefer |
| Brace init | `Dog d{"Rex"}` | Stack | Prevent narrowing, uniform style |
| Value init | `Dog d{}` | Stack | Default-construct / zero-init |
| Copy init | `Dog d = other` | Stack | Copy from existing object |
| Move init | `Dog d = move(o)` | Stack | Transfer ownership cheaply |
| Raw `new` | `new Dog("Rex")` | Heap | Low-level code only — avoid |
| `make_unique` | `make_unique<Dog>("Rex")` | Heap | Sole ownership |
| `make_shared` | `make_shared<Dog>("Rex")` | Heap | Shared ownership |
| Placement `new` | `new(buf) Dog("Rex")` | Custom | Memory pools, no-alloc zones |
| `construct_at` | `construct_at(ptr, "Rex")` | Custom | C++20, constexpr placement |
| `emplace_back` | `vec.emplace_back("Rex")` | Container | In-container, no temp |
| Aggregate init | `Point{.x=1, .y=2}` | Stack / any | Plain structs |
| Factory fn | `makeShape("circle")` | Any | Hide ctor, named creation |
| Named ctor | `Angle::fromDegrees(90)` | Stack | Disambiguate overloads |
| `clone()` | `base->clone()` | Heap | Polymorphic copy |
| `create()` | `base->create()` | Heap | Polymorphic default ctor |
| Static local | `static Dog d` | Static | Singleton, one-time init |
| `optional::emplace` | `opt.emplace("Rex")` | Optional | Nullable, in-place |
| Object pool | `pool.acquire("Rex")` | Pre-alloc | High-frequency, no GC |
| Deserialization | `Dog::fromJson(j)` | Any | Reconstruct from data |

## How to Choose

```
Do you know the type at compile time?
│
├── YES — Does it need to outlive its scope?
│         │
│         ├── NO  → Stack: Dog d{"Rex"}              ← always prefer this
│         │
│         └── YES → Single owner?
│                   ├── YES → make_unique<Dog>("Rex")
│                   └── NO  → make_shared<Dog>("Rex")
│
└── NO  — Do you need polymorphism?
          │
          ├── YES → Virtual clone() / factory function / registry
          │
          └── NO  → std::variant (closed set) / std::any (open set)

Special cases:
  Constructing into existing memory  →  placement new / construct_at
  Creating in a container            →  emplace_back / emplace
  One instance globally              →  static local (Meyers singleton)
  High-frequency, no heap            →  object pool + placement new
  Nullable object                    →  std::optional + emplace
  Disambiguating constructors        →  named constructor (static factory)
```
