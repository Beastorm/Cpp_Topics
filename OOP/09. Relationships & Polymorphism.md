## Composition vs Aggregation in C++
Both represent "has-a" relationships, but differ in ownership semantics and lifecycle management.

**Overview:**
``` 
┌─────────────────────────────────────────────────────────────────────┐
│                    HAS-A RELATIONSHIPS                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  COMPOSITION                    AGGREGATION                         │
│  "Owns-a" / "Part-of"          "Uses-a" / "Has-reference-to"       │
│                                                                     │
│  ◆━━━━━━━━━━━━━━━━━━━━◆        ◇━━━━━━━━━━━━━━━━━━━━◇              │
│  Strong ownership              Weak ownership                       │
│  Part dies with whole          Part can exist independently         │
│  Exclusive relationship        Shared relationship                  │
│                                                                     │
│  Example:                      Example:                             │
│  Car ◆━━━ Engine              University ◇━━━ Student              │
│  (Engine can't exist           (Students exist independently)       │
│   without Car)                                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 1. COMPOSITION (Strong Ownership)
**Definition**: The contained object is part of the container and cannot exist independently.

**Characteristics:**
✅ Strong lifecycle dependency: Part is created and destroyed with the whole
✅ Exclusive ownership: Part belongs to exactly one owner
✅ Implemented with: Direct member objects or unique_ptr
✅ UML: Filled diamond (◆)

**Example 1:** Direct Member Objects

``` cpp
#include <iostream>
#include <string>
using namespace std;

class Engine {
private:
    int horsepower;
public:
    Engine(int hp) : horsepower(hp) {
        cout << "Engine created (" << hp << " HP)" << endl;
    }
    ~Engine() {
        cout << "Engine destroyed" << endl;
    }
    void start() {
        cout << "Engine started (" << horsepower << " HP)" << endl;
    }
};

class Wheel {
private:
    int size;
public:
    Wheel(int s) : size(s) {
        cout << "Wheel created (" << s << "\")" << endl;
    }
    ~Wheel() {
        cout << "Wheel destroyed" << endl;
    }
};

// COMPOSITION: Car owns Engine and Wheels
class Car {
private:
    string model;
    Engine engine;      // Composition: Engine is part of Car
    Wheel wheels[4];    // Composition: Wheels are part of Car

public:
    // Engine and Wheels are created when Car is created
    Car(string m, int hp, int wheelSize) 
        : model(m), 
          engine(hp),
          wheels{Wheel(wheelSize), Wheel(wheelSize), 
                 Wheel(wheelSize), Wheel(wheelSize)} {
        cout << "Car created: " << model << endl;
    }
    
    // Engine and Wheels are destroyed when Car is destroyed
    ~Car() {
        cout << "Car destroyed: " << model << endl;
    }
    
    void start() {
        cout << model << " starting..." << endl;
        engine.start();
    }
};

int main() {
    {
        Car myCar("Tesla Model 3", 450, 18);
        myCar.start();
    }  // Car destroyed -> Engine and Wheels automatically destroyed
    
    cout << "\nProgram ending..." << endl;
    return 0;
}
```

**Output:**

```text

Engine created (450 HP)
Wheel created (18")
Wheel created (18")
Wheel created (18")
Wheel created (18")
Car created: Tesla Model 3
Tesla Model 3 starting...
Engine started (450 HP)
Car destroyed: Tesla Model 3
Engine destroyed
Wheel destroyed
Wheel destroyed
Wheel destroyed
Wheel destroyed

Program ending...
```

**Example 2:** Composition with unique_ptr

``` cpp
#include <iostream>
#include <memory>
#include <string>
using namespace std;

class Brain {
private:
    int iq;
public:
    Brain(int intelligence) : iq(intelligence) {
        cout << "Brain created (IQ: " << iq << ")" << endl;
    }
    ~Brain() {
        cout << "Brain destroyed" << endl;
    }
    void think() {
        cout << "Thinking with IQ " << iq << endl;
    }
};

class Heart {
private:
    int bpm;
public:
    Heart(int rate) : bpm(rate) {
        cout << "Heart created (BPM: " << bpm << ")" << endl;
    }
    ~Heart() {
        cout << "Heart destroyed" << endl;
    }
    void beat() {
        cout << "Heart beating at " << bpm << " BPM" << endl;
    }
};

// COMPOSITION: Person owns Brain and Heart
class Person {
private:
    string name;
    unique_ptr<Brain> brain;   // Exclusive ownership
    unique_ptr<Heart> heart;   // Exclusive ownership

public:
    Person(string n, int iq, int bpm) : name(n) {
        brain = make_unique<Brain>(iq);
        heart = make_unique<Heart>(bpm);
        cout << "Person created: " << name << endl;
    }
    
    ~Person() {
        cout << "Person destroyed: " << name << endl;
        // brain and heart automatically deleted
    }
    
    // Delete copy (composition objects can't be shared)
    Person(const Person&) = delete;
    Person& operator=(const Person&) = delete;
    
    // Move is OK (ownership transfer)
    Person(Person&&) = default;
    Person& operator=(Person&&) = default;
    
    void live() {
        brain->think();
        heart->beat();
    }
};

int main() {
    {
        Person alice("Alice", 130, 70);
        alice.live();
        
        // Person bob = alice;  // ❌ ERROR: Can't copy
        Person bob = std::move(alice);  // ✅ OK: Move ownership
        bob.live();
        
    }  // Person destroyed -> Brain and Heart destroyed
    
    return 0;
}
```
**Output:**

```text

Brain created (IQ: 130)
Heart created (BPM: 70)
Person created: Alice
Thinking with IQ 130
Heart beating at 70 BPM
Thinking with IQ 130
Heart beating at 70 BPM
Person destroyed: Alice
Heart destroyed
Brain destroyed
```

### 2. AGGREGATION (Weak Ownership)
**Definition**: The container uses objects that can exist independently.

**Characteristics:**
✅ Independent lifecycle: Parts can exist before/after the whole
✅ Shared ownership: Parts can be shared by multiple containers
✅ Implemented with: Pointers, references, or shared_ptr
✅ UML: Hollow diamond (◇)


**Example 1:** Aggregation with Raw Pointers

``` cpp
#include <iostream>
#include <vector>
#include <string>
using namespace std;

class Student {
private:
    string name;
    int id;
public:
    Student(string n, int i) : name(n), id(i) {
        cout << "Student created: " << name << endl;
    }
    ~Student() {
        cout << "Student destroyed: " << name << endl;
    }
    string getName() const { return name; }
    int getId() const { return id; }
};

// AGGREGATION: Department uses Students (doesn't own them)
class Department {
private:
    string deptName;
    vector<Student*> students;  // Non-owning pointers

public:
    Department(string name) : deptName(name) {
        cout << "Department created: " << deptName << endl;
    }
    
    ~Department() {
        cout << "Department destroyed: " << deptName << endl;
        // Does NOT delete students (doesn't own them)
    }
    
    void addStudent(Student* student) {
        students.push_back(student);
        cout << deptName << " added student: " 
             << student->getName() << endl;
    }
    
    void removeStudent(int id) {
        students.erase(
            remove_if(students.begin(), students.end(),
                [id](Student* s) { return s->getId() == id; }),
            students.end()
        );
    }
    
    void listStudents() const {
        cout << deptName << " students:" << endl;
        for (const auto* s : students) {
            cout << "  - " << s->getName() << endl;
        }
    }
};

int main() {
    // Students exist independently
    Student alice("Alice", 1);
    Student bob("Bob", 2);
    Student charlie("Charlie", 3);
    
    {
        Department csDept("Computer Science");
        Department mathDept("Mathematics");
        
        // Students can be in multiple departments (aggregation)
        csDept.addStudent(&alice);
        csDept.addStudent(&bob);
        
        mathDept.addStudent(&bob);     // Bob in both departments
        mathDept.addStudent(&charlie);
        
        csDept.listStudents();
        mathDept.listStudents();
        
    }  // Departments destroyed, but Students still exist!
    
    cout << "\nStudents still alive after departments destroyed" << endl;
    
    return 0;
}  // Students destroyed here
```
**Output:**

```text

Student created: Alice
Student created: Bob
Student created: Charlie
Department created: Computer Science
Department created: Mathematics
Computer Science added student: Alice
Computer Science added student: Bob
Mathematics added student: Bob
Mathematics added student: Charlie
Computer Science students:
  - Alice
  - Bob
Mathematics students:
  - Bob
  - Charlie
Department destroyed: Mathematics
Department destroyed: Computer Science

Students still alive after departments destroyed
Student destroyed: Charlie
Student destroyed: Bob
Student destroyed: Alice
```
**Example 2:** Aggregation with shared_ptr

``` cpp
#include <iostream>
#include <memory>
#include <vector>
#include <string>
using namespace std;

class Book {
private:
    string title;
    string isbn;
public:
    Book(string t, string i) : title(t), isbn(i) {
        cout << "Book created: " << title << endl;
    }
    ~Book() {
        cout << "Book destroyed: " << title << endl;
    }
    string getTitle() const { return title; }
    string getISBN() const { return isbn; }
};

// AGGREGATION: Library shares ownership of Books
class Library {
private:
    string name;
    vector<shared_ptr<Book>> books;  // Shared ownership

public:
    Library(string n) : name(n) {
        cout << "Library created: " << name << endl;
    }
    
    ~Library() {
        cout << "Library destroyed: " << name << endl;
        // Books destroyed only if no other references exist
    }
    
    void addBook(shared_ptr<Book> book) {
        books.push_back(book);
        cout << name << " added book: " << book->getTitle() 
             << " (ref count: " << book.use_count() << ")" << endl;
    }
    
    void listBooks() const {
        cout << name << " collection:" << endl;
        for (const auto& book : books) {
            cout << "  - " << book->getTitle() 
                 << " (ref count: " << book.use_count() << ")" << endl;
        }
    }
};

class Person {
private:
    string name;
    vector<shared_ptr<Book>> ownedBooks;  // Shared ownership

public:
    Person(string n) : name(n) {
        cout << "Person created: " << name << endl;
    }
    
    ~Person() {
        cout << "Person destroyed: " << name << endl;
    }
    
    void borrowBook(shared_ptr<Book> book) {
        ownedBooks.push_back(book);
        cout << name << " borrowed: " << book->getTitle()
             << " (ref count: " << book.use_count() << ")" << endl;
    }
    
    void returnBook(const string& isbn) {
        ownedBooks.erase(
            remove_if(ownedBooks.begin(), ownedBooks.end(),
                [&isbn](const shared_ptr<Book>& b) { 
                    return b->getISBN() == isbn; 
                }),
            ownedBooks.end()
        );
        cout << name << " returned book " << isbn << endl;
    }
};

int main() {
    auto book1 = make_shared<Book>("C++ Primer", "123");
    auto book2 = make_shared<Book>("Effective C++", "456");
    
    {
        Library lib("City Library");
        lib.addBook(book1);
        lib.addBook(book2);
        
        {
            Person reader("Alice");
            reader.borrowBook(book1);  // Shared with library
            
            lib.listBooks();
            
            reader.returnBook("123");
        }  // Person destroyed, book1 ref count decreases
        
        lib.listBooks();
        
    }  // Library destroyed, but books may still exist
    
    cout << "\nbook1 ref count: " << book1.use_count() << endl;
    cout << "book2 ref count: " << book2.use_count() << endl;
    
    return 0;
}  // Books finally destroyed when last reference goes away
```

**Output:**
```
Book created: C++ Primer
Book created: Effective C++
Library created: City Library
City Library added book: C++ Primer (ref count: 2)
City Library added book: Effective C++(ref count: 2)
Person created: Alice
Alice borrowed: C++ Primer (ref count: 3)
City Library collection:
  - C++ Primer (ref count: 3)
  - Effective C++ (ref count: 2)
Alice returned book 123
Person destroyed: Alice
City Library collection:
  - C++ Primer (ref count: 2)
  - Effective C++ (ref count: 2)
Library destroyed: City Library

book1 ref count: 1
book2 ref count: 1
Book destroyed: Effective C++
Book destroyed: C++ Primer
```

### 3. Side-by-Side Comparison
**Implementation Comparison:**

``` cpp
#include <iostream>
#include <memory>
using namespace std;

class Component {
public:
    Component() { cout << "Component created" << endl; }
    ~Component() { cout << "Component destroyed" << endl; }
    void operate() { cout << "Component operating" << endl; }
};

// ═══════════════════════════════════════════════════════════
// COMPOSITION
// ═══════════════════════════════════════════════════════════
class CompositionExample {
private:
    Component comp;  // Owned member - created/destroyed with owner

public:
    CompositionExample() : comp() {
        cout << "CompositionExample created" << endl;
    }
    
    ~CompositionExample() {
        cout << "CompositionExample destroyed" << endl;
    }
    
    void use() {
        comp.operate();
    }
};

// ═══════════════════════════════════════════════════════════
// COMPOSITION with unique_ptr
// ═══════════════════════════════════════════════════════════
class CompositionUnique {
private:
    unique_ptr<Component> comp;  // Exclusive ownership

public:
    CompositionUnique() {
        comp = make_unique<Component>();
        cout << "CompositionUnique created" << endl;
    }
    
    ~CompositionUnique() {
        cout << "CompositionUnique destroyed" << endl;
    }
    
    void use() {
        comp->operate();
    }
};

// ═══════════════════════════════════════════════════════════
// AGGREGATION with shared_ptr
// ═══════════════════════════════════════════════════════════
class AggregationShared {
private:
    shared_ptr<Component> comp;  // Shared ownership

public:
    AggregationShared(shared_ptr<Component> c) : comp(c) {
        cout << "AggregationShared created (ref count: " 
             << comp.use_count() << ")" << endl;
    }
    
    ~AggregationShared() {
        cout << "AggregationShared destroyed (ref count: " 
             << comp.use_count() << ")" << endl;
    }
    
    void use() {
        comp->operate();
    }
};

// ═══════════════════════════════════════════════════════════
// AGGREGATION with raw pointer
// ═══════════════════════════════════════════════════════════
class AggregationRaw {
private:
    Component* comp;  // Non-owning pointer

public:
    AggregationRaw(Component* c) : comp(c) {
        cout << "AggregationRaw created" << endl;
    }
    
    ~AggregationRaw() {
        cout << "AggregationRaw destroyed (NOT deleting component)" << endl;
    }
    
    void use() {
        if (comp) comp->operate();
    }
};

int main() {
    cout << "=== COMPOSITION (direct member) ===" << endl;
    {
        CompositionExample obj;
        obj.use();
    }
    cout << endl;
    
    cout << "=== COMPOSITION (unique_ptr) ===" << endl;
    {
        CompositionUnique obj;
        obj.use();
    }
    cout << endl;
    
    cout << "=== AGGREGATION (shared_ptr) ===" << endl;
    {
        auto sharedComp = make_shared<Component>();
        {
            AggregationShared obj1(sharedComp);
            AggregationShared obj2(sharedComp);
            obj1.use();
        }  // Objects destroyed, but Component still alive
        cout << "After objects destroyed, ref count: " 
             << sharedComp.use_count() << endl;
    }  // Component destroyed when last reference goes
    cout << endl;
    
    cout << "=== AGGREGATION (raw pointer) ===" << endl;
    {
        Component externalComp;
        {
            AggregationRaw obj(&externalComp);
            obj.use();
        }  // Object destroyed, Component still alive
        cout << "Component still exists" << endl;
    }  // Component destroyed here
    
    return 0;
}
```

### 4. Decision Guide

``` 
┌─────────────────────────────────────────────────────────────────────┐
│                    WHEN TO USE WHICH?                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  USE COMPOSITION WHEN:                                              │
│  ✓ Part has no meaning without the whole                           │
│  ✓ Part should be destroyed when whole is destroyed                │
│  ✓ Part belongs to exactly ONE owner                               │
│  ✓ Strong coupling is acceptable                                   │
│                                                                     │
│  Examples:                                                          │
│    • Car and Engine                                                 │
│    • House and Rooms                                                │
│    • Person and Brain                                               │
│    • File and FileHandle                                            │
│                                                                     │
│  Implementation: Direct members or unique_ptr                       │
│                                                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  USE AGGREGATION WHEN:                                              │
│  ✓ Part can exist independently                                    │
│  ✓ Part can be shared by multiple containers                       │
│  ✓ Part's lifetime is managed elsewhere                            │
│  ✓ Loose coupling is desired                                       │
│                                                                     │
│  Examples:                                                          │
│    • University and Students                                        │
│    • Library and Books                                              │
│    • Team and Members                                               │
│    • Playlist and Songs                                             │
│                                                                     │
│  Implementation: Pointers, references, or shared_ptr                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 5. Real-World Complete Example
```cpp
#include <iostream>
#include <memory>
#include <vector>
#include <string>
using namespace std;

// ═══════════════════════════════════════════════════════════
// Components (can exist independently)
// ═══════════════════════════════════════════════════════════
class Processor {
private:
    string model;
    int cores;
public:
    Processor(string m, int c) : model(m), cores(c) {}
    string getInfo() const { 
        return model + " (" + to_string(cores) + " cores)"; 
    }
};

// ═══════════════════════════════════════════════════════════
// Parts (composition - can't exist without owner)
// ═══════════════════════════════════════════════════════════
class Motherboard {
private:
    string model;
public:
    Motherboard(string m) : model(m) {
        cout << "  Motherboard initialized: " << model << endl;
    }
    ~Motherboard() {
        cout << "  Motherboard destroyed: " << model << endl;
    }
};

class PowerSupply {
private:
    int wattage;
public:
    PowerSupply(int w) : wattage(w) {
        cout << "  PSU initialized: " << wattage << "W" << endl;
    }
    ~PowerSupply() {
        cout << "  PSU destroyed" << endl;
    }
};

// ═══════════════════════════════════════════════════════════
// COMPOSITION + AGGREGATION Example
// ═══════════════════════════════════════════════════════════
class Computer {
private:
    string name;
    
    // COMPOSITION: These are parts OF the computer
    unique_ptr<Motherboard> motherboard;  // Exclusive ownership
    unique_ptr<PowerSupply> psu;          // Exclusive ownership
    
    // AGGREGATION: Computer USES a processor (can be shared/reused)
    shared_ptr<Processor> processor;      // Shared ownership

public:
    Computer(string n, shared_ptr<Processor> cpu)
        : name(n), processor(cpu) {
        
        cout << "Building computer: " << name << endl;
        
        // Create owned parts (composition)
        motherboard = make_unique<Motherboard>("ASUS ROG");
        psu = make_unique<PowerSupply>(750);
        
        cout << "  CPU: " << processor->getInfo() << endl;
        cout << "Computer built: " << name << endl;
        cout << endl;
    }
    
    ~Computer() {
        cout << "Destroying computer: " << name << endl;
        // motherboard and psu automatically destroyed
        // processor reference count decreases
    }
    
    void displaySpecs() const {
        cout << name << " specs:" << endl;
        cout << "  Processor: " << processor->getInfo() << endl;
        cout << "  Processor ref count: " << processor.use_count() << endl;
    }
};

int main() {
    // Create processors (exist independently)
    auto intelCPU = make_shared<Processor>("Intel i9", 16);
    auto amdCPU = make_shared<Processor>("AMD Ryzen 9", 12);
    
    cout << "=== Creating Computers ===" << endl;
    {
        // Multiple computers can share same processor type (aggregation)
        Computer pc1("Gaming Rig", intelCPU);
        Computer pc2("Workstation", intelCPU);  // Shares same CPU reference
        Computer pc3("Server", amdCPU);
        
        pc1.displaySpecs();
        pc2.displaySpecs();
        pc3.displaySpecs();
        
        cout << "\n=== Computers going out of scope ===" << endl;
    }  // All computers destroyed (motherboards and PSUs destroyed)
       // But processors still exist!
    
    cout << "\n=== Processors still alive ===" << endl;
    cout << "Intel ref count: " << intelCPU.use_count() << endl;
    cout << "AMD ref count: " << amdCPU.use_count() << endl;
    
    return 0;
}  // Processors finally destroyed
```

**Key Takeaway:**
Composition = "Cannot exist without" → Use direct members or unique_ptr
Aggregation = "Can exist independently" → Use pointers or shared_ptr
Choose based on ownership semantics and lifecycle requirements! 🎯

## Overriding vs Overloading in C++
Two different concepts that are often confused — both involve multiple functions with the same name, but serve completely different purposes.

**Quick Comparison:**
```
┌─────────────────────────────────────────────────────────────────────┐
│                  OVERLOADING vs OVERRIDING                          │
├────────────────────────────┬────────────────────────────────────────┤
│      OVERLOADING           │          OVERRIDING                    │
├────────────────────────────┼────────────────────────────────────────┤
│ Same name, different       │ Same name, same signature              │
│ parameters                 │                                        │
│                            │                                        │
│ COMPILE-TIME polymorphism  │ RUNTIME polymorphism                   │
│ (Static binding)           │ (Dynamic binding)                      │
│                            │                                        │
│ Same class or scope        │ Base class + Derived class             │
│                            │                                        │
│ No 'virtual' needed        │ Requires 'virtual' in base             │
│                            │                                        │
│ Parameters MUST differ     │ Parameters MUST be identical           │
│                            │                                        │
│ Return type can differ     │ Return type must be same/covariant     │
│                            │                                        │
│ Resolved at compile time   │ Resolved at runtime via vtable         │
└────────────────────────────┴────────────────────────────────────────┘
```

### 1. FUNCTION OVERLOADING
**Definition:** Multiple functions with the same name but different parameters in the same scope.

**Basic Examples:**
``` cpp
#include <iostream>
#include <string>
using namespace std;

class Calculator {
public:
    // Overloaded by number of parameters
    int add(int a, int b) {
        cout << "add(int, int)" << endl;
        return a + b;
    }
    
    int add(int a, int b, int c) {
        cout << "add(int, int, int)" << endl;
        return a + b + c;
    }
    
    // Overloaded by type of parameters
    double add(double a, double b) {
        cout << "add(double, double)" << endl;
        return a + b;
    }
    
    // Overloaded by parameter order
    void display(int num, string text) {
        cout << "display(int, string): " << num << ", " << text << endl;
    }
    
    void display(string text, int num) {
        cout << "display(string, int): " << text << ", " << num << endl;
    }
};

int main() {
    Calculator calc;
    
    cout << calc.add(5, 10) << endl;           // Calls add(int, int)
    cout << calc.add(5, 10, 15) << endl;       // Calls add(int, int, int)
    cout << calc.add(3.14, 2.86) << endl;      // Calls add(double, double)
    
    calc.display(42, "Answer");                 // display(int, string)
    calc.display("Answer", 42);                 // display(string, int)
    
    return 0;
}
```

**Output:**


### 2. FUNCTION OVERRIDING
**Definition:** Redefining a virtual function in a derived class with the same signature to provide a specific implementation.

**Basic Example:**
``` CPP
#include <iostream>
#include <string>
using namespace std;

class Animal {
public:
    // Virtual function enables overriding
    virtual void makeSound() const {
        cout << "Animal makes a generic sound" << endl;
    }
    
    virtual void eat() const {
        cout << "Animal eats food" << endl;
    }
    
    // Virtual destructor (important for polymorphism)
    virtual ~Animal() {
        cout << "Animal destructor" << endl;
    }
};

class Dog : public Animal {
public:
    // Override base class function
    void makeSound() const override {  // 'override' keyword (C++11)
        cout << "Dog barks: Woof! Woof!" << endl;
    }
    
    void eat() const override {
        cout << "Dog eats dog food" << endl;
    }
    
    ~Dog() {
        cout << "Dog destructor" << endl;
    }
};

class Cat : public Animal {
public:
    void makeSound() const override {
        cout << "Cat meows: Meow! Meow!" << endl;
    }
    
    void eat() const override {
        cout << "Cat eats cat food" << endl;
    }
    
    ~Cat() {
        cout << "Cat destructor" << endl;
    }
};

int main() {
    // Runtime polymorphism
    Animal* animals[3];
    animals[0] = new Animal();
    animals[1] = new Dog();
    animals[2] = new Cat();
    
    cout << "=== Polymorphic behavior ===" << endl;
    for (int i = 0; i < 3; i++) {
        animals[i]->makeSound();  // Calls appropriate version at runtime
        animals[i]->eat();
        cout << endl;
    }
    
    cout << "=== Cleanup ===" << endl;
    for (int i = 0; i < 3; i++) {
        delete animals[i];  // Virtual destructor ensures proper cleanup
    }
    
    return 0;
}
```

``` cpp
};
```

``` cpp
private:
    double width, height;

public:

    double area() const override {
        return width * height;
    }

        return 2 * (width + height);

        Shape::display();  // Call base version
        cout << "  Width: " << width << ", Height: " << height << endl;
        cout << "  Area: " << area() << ", Perimeter: " << perimeter() << endl;
    }
};

class Circle : public Shape {
private:
    double radius;

public:
    Circle(double r) : Shape("Circle"), radius(r) { }

        return PI * radius * radius;
    }
    
    double perimeter() const override {
        return 2 * PI * radius;
    }
    
    void display() const override {
        Shape::display();
        cout << "  Radius: " << radius << endl;
        cout << "  Area: " << area() << ", Perimeter: " << perimeter() << endl;
    }
};

    }
```

### 3. Virtual Table (vtable) Mechanism
``` cpp
#include <iostream>
using namespace std;

class Base {
public:
    virtual void vfunc1() { cout << "Base::vfunc1()" << endl; }
    virtual void vfunc2() { cout << "Base::vfunc2()" << endl; }
    void regularFunc() { cout << "Base::regularFunc()" << endl; }
};

class Derived : public Base {
public:
    void vfunc1() override { cout << "Derived::vfunc1()" << endl; }
    // vfunc2 not overridden
    void regularFunc() { cout << "Derived::regularFunc()" << endl; }
};

int main() {
    Base* ptr;
    Derived d;
    ptr = &d;
    
    cout << "=== Virtual functions (runtime binding) ===" << endl;
    ptr->vfunc1();        // Derived::vfunc1() - via vtable
    ptr->vfunc2();        // Base::vfunc2() - via vtable
    
    cout << "\n=== Non-virtual function (compile-time binding) ===" << endl;
    ptr->regularFunc();   // Base::regularFunc() - static binding!
    
    cout << "\n=== Direct object call ===" << endl;
    d.vfunc1();           // Derived::vfunc1()
    d.regularFunc();      // Derived::regularFunc()
    
    return 0;
}
```

**Output:**

```text

=== Virtual functions (runtime binding) ===
Derived::vfunc1()
Base::vfunc2()

=== Non-virtual function (compile-time binding) ===
Base::regularFunc()

=== Direct object call ===
Derived::vfunc1()
Derived::regularFunc()
```

### 5. Common Pitfalls
``` cpp
class Base {
public:
    virtual void func(int x) {
        cout << "Base::func(int)" << endl;
    }
};

class Derived : public Base {
public:
    // ❌ PITFALL: This is NOT overriding, it's HIDING!
    // (different parameter type)
    void func(double x) {
        cout << "Derived::func(double)" << endl;
    }
};

int main() {
    Derived d;
    d.func(10);      // Calls Derived::func(double) - int converted to double!
    d.func(10.5);    // Calls Derived::func(double)
    
    // d.func(10) does NOT call Base::func(int)!
    // Base version is hidden, not overridden
    
    // To call base version:
    d.Base::func(10);  // Explicitly call Base::func(int)
    
    return 0;
}
```
**Fix with using declaration:**
``` cpp
class Derived : public Base {
public:
    using Base::func;  // Bring base versions into scope
    
    void func(double x) {
        cout << "Derived::func(double)" << endl;
    }
};

int main() {
    Derived d;
    d.func(10);      // Now calls Base::func(int)
    d.func(10.5);    // Calls Derived::func(double)
    return 0;
}
```

**Summary Table:**
``` 
┌────────────────────────────────────────────────────────────────────┐
│                     DECISION GUIDE                                 │
├────────────────────────────────────────────────────────────────────┤
│                                                                    │
│  USE OVERLOADING WHEN:                                             │
│  ✓ Same operation, different input types                          │
│  ✓ Providing convenience methods                                  │
│  ✓ Compile-time resolution is sufficient                          │
│  ✓ Working within same class                                      │
│                                                                    │
│  Examples: Constructors, operators, print(), add(), sort()        │
│                                                                    │
├────────────────────────────────────────────────────────────────────┤
│                                                                    │
│  USE OVERRIDING WHEN:                                              │
│  ✓ Specialized behavior in derived classes                        │
│  ✓ Runtime polymorphism needed                                    │
│  ✓ Interface implementation (pure virtual)                        │
│  ✓ Working across inheritance hierarchy                           │
│                                                                    │
│  Examples: draw(), makeSound(), calculate(), serialize()          │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```

### Key Insight:
Overloading = Multiple implementations at compile time (static)
Overriding = Specific implementation chosen at runtime (dynamic)
Both are essential tools in C++ polymorphism! 🎯

## Interview Questions: Overriding vs Overloading
🎯 Beginner Level Questions
### Q1: What's the difference between function overloading and function overriding?
**Answer:**
``` cpp
// OVERLOADING: Same name, different parameters, SAME CLASS
class Calculator {
public:
    int add(int a, int b) { return a + b; }           // 2 params
    int add(int a, int b, int c) { return a + b + c; } // 3 params
    double add(double a, double b) { return a + b; }   // different type
};
// Resolved at COMPILE-TIME

// OVERRIDING: Same signature, DIFFERENT CLASSES (inheritance)
class Base {
public:
    virtual void show() { cout << "Base"; }
};

class Derived : public Base {
public:
    void show() override { cout << "Derived"; }  // Same signature
};
// Resolved at RUNTIME
```
**Key Points:**
Overloading: Compile-time polymorphism, same class, different signatures
Overriding: Runtime polymorphism, inheritance hierarchy, same signature


### Q2: Can you override a non-virtual function?
**Answer:**
``` cpp
class Base {
public:
    void display() {  // NOT virtual
        cout << "Base::display()" << endl;
    }
};

class Derived : public Base {
public:
    void display() {  // This is HIDING, not overriding!
        cout << "Derived::display()" << endl;
    }
};

int main() {
    Derived d;
    Base* ptr = &d;
    
    ptr->display();  // Outputs: "Base::display()"
                     // NOT polymorphic - calls Base version!
    
    d.display();     // Outputs: "Derived::display()"
    
    return 0;
}
```
**Answer:** No, you cannot override non-virtual functions. Without virtual, you get function hiding (name shadowing), not overriding. The function called depends on the pointer type, not the object type.

### Q3: What happens if you forget the override keyword?
**Answer:**
``` cpp
class Base {
public:
    virtual void func(int x) const {
        cout << "Base::func" << endl;
    }
};

class Derived : public Base {
public:
    // Missing 'const' - signature doesn't match!
    void func(int x) {  // No override keyword
        cout << "Derived::func" << endl;
    }
};

int main() {
    Base* ptr = new Derived();
    ptr->func(10);  // Calls Base::func - NOT what you expected!
    delete ptr;
    return 0;
}
```
**Answer:** Without override, the compiler won't catch signature mismatches. The code compiles but doesn't work as intended. The override keyword (C++11) forces compile-time verification that you're actually overriding a base class function.

### Q5: Can you overload based on return type alone?
**Answer:**

``` cpp
class Test {
```

### Q6: What is function hiding? How is it different from overriding?
**Answer:**
``` cpp
class Base {
public:
    virtual void func(int x) {
        cout << "Base::func(int)" << endl;
    }
    
    void display() {
        cout << "Base::display()" << endl;
    }
};

class Derived : public Base {
public:
    // HIDING: Same name, different signature
    void func(double x) {  // Hides ALL Base::func versions
        cout << "Derived::func(double)" << endl;
    }
    
    // HIDING: Non-virtual function with same signature
    void display() {  // Hides Base::display()
        cout << "Derived::display()" << endl;
    }
};

int main() {
    Derived d;
    d.func(10);      // Calls Derived::func(double) - 10 converted to 10.0
    // d.func(10) does NOT call Base::func(int) - it's hidden!
    
    Base* ptr = &d;
    ptr->func(10);   // Calls Base::func(int) - not virtual override
    ptr->display();  // Calls Base::display() - not virtual
    
    return 0;
}
```

Comparison: Overriding vs. Hiding

| Feature | Overriding | Hiding (Shadowing) |
| :--- | :--- | :--- |
| **Virtual keyword** | Required | Not required |
| **Signature** | Must be identical | Can be different |
| **Polymorphism** | Runtime (dynamic) | Compile-time (static) |
| **Scope** | Replaces base function | Shadows base function |

### Q7: Explain the vtable mechanism. What's the performance cost?
**Answer:**
``` cpp
#include <iostream>
using namespace std;

class Base {
public:
    virtual void v1() { cout << "Base::v1" << endl; }
    virtual void v2() { cout << "Base::v2" << endl; }
    void nonVirtual() { cout << "Base::nonVirtual" << endl; }
};

class Derived : public Base {
public:
    void v1() override { cout << "Derived::v1" << endl; }
    // v2 not overridden
};

int main() {
    cout << "Size of Base: " << sizeof(Base) << " bytes" << endl;
    // Includes vptr (typically 8 bytes on 64-bit system)
    
    Base* ptr = new Derived();
    
    // Virtual call: ptr->vptr->vtable[index]->function
    ptr->v1();  // Indirect call through vtable (small overhead)
    
    // Non-virtual call: Direct call (no overhead)
    ptr->nonVirtual();
    
    delete ptr;
    return 0;
}
```
**Memory Layout:**

``` 
Base Object:
┌─────────────┐
│  vptr       │ ──→ Base vtable: [&Base::v1, &Base::v2]
└─────────────┘

Derived Object:
┌─────────────┐
│  vptr       │ ──→ Derived vtable: [&Derived::v1, &Base::v2]
└─────────────┘
```

**Performance Cost:**
- Memory: Extra pointer (vptr) per object with virtual functions (~8 bytes)
- Time: One extra indirection per virtual call (negligible in most cases)
- Cache: Vtable lookup may cause a cache miss

**When to avoid virtual functions:**
- Real-time systems with strict performance requirements
- Tiny, frequently-created objects
- Hot paths in performance-critical code

## 💎 Advanced Level Questions
### Q8: What's wrong with this code?

``` cpp
class Base {
public:
    Base() { init(); }
    virtual void init() { cout << "Base init" << endl; }
    virtual ~Base() { cleanup(); }
    virtual void cleanup() { cout << "Base cleanup" << endl; }
};

class Derived : public Base {
private:
    int* data;
public:
    Derived() : data(new int[100]) {
        cout << "Derived constructor" << endl;
    }
    
    void init() override {
        cout << "Derived init" << endl;
        data[0] = 42;  // Using data
    }
    
    void cleanup() override {
        cout << "Derived cleanup" << endl;
        delete[] data;
    }
    
    ~Derived() {
        cout << "Derived destructor" << endl;
    }
};

int main() {
    Derived d;
    return 0;
}
```

**Answer:**
**Problem 1:** Calling virtual functions in constructor
During Base constructor, the object is still a Base, not Derived
init() will call Base::init(), not Derived::init()
data is not initialized yet when Derived::init() would be called   


**Problem 2:** Calling virtual functions in destructor
During ~Base(), the object is already back to being a Base
cleanup() calls Base::cleanup(), not Derived::cleanup()
Memory leak! data is never deleted

**Output:**
```
Base init
Derived constructor
Derived destructor
Base cleanup        // Wrong! Should be Derived cleanup
```


**Fix:**
``` cpp
class Base {
public:
    Base() { /* Don't call virtual functions */ }
    virtual ~Base() { /* Don't call virtual functions */ }
};

class Derived : public Base {
private:
    int* data;
public:
    Derived() : data(new int[100]) {
        init();  // Call explicitly, not through virtual dispatch
    }
    
    ~Derived() {
        cleanup();  // Call explicitly before destruction
    }
    
    void init() { data[0] = 42; }
    void cleanup() { delete[] data; }
};
```

### Q9: Implement a type-safe function overloading that prevents implicit conversions
**Question:** Create an add function that only accepts exactly matching types.

**Answer:**
``` cpp
#include <iostream>
#include <type_traits>
using namespace std;

class Calculator {
public:
    // Overload for int
    int add(int a, int b) {
        cout << "add(int, int)" << endl;
        return a + b;
    }
    
    // Overload for double
    double add(double a, double b) {
        cout << "add(double, double)" << endl;
        return a + b;
    }
    
    // Template that catches any other combination
    template<typename T, typename U>
    auto add(T a, U b) -> typename enable_if<
        !is_same<T, int>::value || !is_same<U, int>::value &&
        !is_same<T, double>::value || !is_same<U, double>::value,
        void
    >::type = delete;  // Delete mismatched types
};

int main() {
    Calculator calc;
    
    calc.add(5, 10);        // ✅ OK: add(int, int)
    calc.add(3.14, 2.86);   // ✅ OK: add(double, double)
    
    // calc.add(5, 3.14);   // ❌ ERROR: Deleted function
    // calc.add(5.0, 10);   // ❌ ERROR: Deleted function
    
    return 0;
}
```

**Question:** Explain and solve the diamond problem.

``` cpp

    virtual void powerOn() { cout << "Device powered on" << endl; }

class Scanner : public Device {
public:
    void powerOn() override { cout << "Scanner powered on" << endl; }

public:

public:
    // Which powerOn() to override?
};

int main() {

    // aio.powerOn();  // ❌ ERROR: Ambiguous!
    
    aio.Scanner::powerOn();  // Must specify which one
```

``` cpp
class Device {

    virtual void powerOn() { cout << "Device powered on" << endl; }
    virtual ~Device() = default;
};

// Virtual inheritance - only ONE Device subobject
class Scanner : virtual public Device {
public:
    void powerOn() override { cout << "Scanner powered on" << endl; }
};

class Printer : virtual public Device {
public:
    void powerOn() override { cout << "Printer powered on" << endl; }
};

class AllInOne : public Scanner, public Printer {
public:
    void powerOn() override {
        cout << "AllInOne powered on" << endl;
        // Can explicitly call base versions if needed
        Scanner::powerOn();
        Printer::powerOn();
    }
};

int main() {
    AllInOne aio;
    aio.powerOn();  // ✅ Works! Calls AllInOne::powerOn()

    ptr->powerOn();      // Calls AllInOne::powerOn()
```

### Q12: Real-world scenario question
**Question:** You're designing a graphics library. You have shapes that can be drawn and need to support both 2D and 3D rendering. How would you use overloading and overriding?

**Answer:**
``` cpp
#include <iostream>
#include <memory>
#include <vector>
using namespace std;

// Renderer interface (Strategy pattern)
class Renderer {
public:
    virtual void renderLine(int x1, int y1, int x2, int y2) = 0;
    virtual void renderCircle(int x, int y, int r) = 0;
    virtual ~Renderer() = default;
};

class Renderer2D : public Renderer {
public:
    void renderLine(int x1, int y1, int x2, int y2) override {
        cout << "2D Line: (" << x1 << "," << y1 << ")->("
             << x2 << "," << y2 << ")" << endl;
    }
    
    void renderCircle(int x, int y, int r) override {
        cout << "2D Circle: center(" << x << "," << y 
             << "), radius=" << r << endl;
    }
};

class Renderer3D : public Renderer {
public:
    void renderLine(int x1, int y1, int x2, int y2) override {
        cout << "3D Line with perspective: (" << x1 << "," << y1 
             << ")->(" << x2 << "," << y2 << ")" << endl;
    }
    
    void renderCircle(int x, int y, int r) override {
        cout << "3D Sphere: center(" << x << "," << y 
             << "), radius=" << r << endl;
    }
};

// Abstract Shape using OVERRIDING
class Shape {
protected:
    Renderer* renderer;
public:
    Shape(Renderer* r) : renderer(r) {}
    virtual void draw() = 0;  // Override in derived classes
    virtual ~Shape() = default;
};

class Circle : public Shape {
private:
    int x, y, radius;
public:
    Circle(Renderer* r, int cx, int cy, int rad) 
        : Shape(r), x(cx), y(cy), radius(rad) {}
    
    void draw() override {
        renderer->renderCircle(x, y, radius);
    }
};

class Line : public Shape {
private:
    int x1, y1, x2, y2;
public:
    Line(Renderer* r, int x1, int y1, int x2, int y2)
        : Shape(r), x1(x1), y1(y1), x2(x2), y2(y2) {}
    
    void draw() override {
        renderer->renderLine(x1, y1, x2, y2);
    }
};

// Graphics context using OVERLOADING
class GraphicsContext {
private:
    vector<unique_ptr<Shape>> shapes;
    Renderer* currentRenderer;

public:
    GraphicsContext(Renderer* r) : currentRenderer(r) {}
    
    // OVERLOADED add methods for different shapes
    void add(int x, int y, int radius) {
        shapes.push_back(make_unique<Circle>(currentRenderer, x, y, radius));
    }
    
    void add(int x1, int y1, int x2, int y2) {
        shapes.push_back(make_unique<Line>(currentRenderer, x1, y1, x2, y2));
    }
    
    void setRenderer(Renderer* r) {
        currentRenderer = r;
        for (auto& shape : shapes) {
            shape->renderer = r;  // Update all shapes
        }
    }
    
    void drawAll() {
        for (auto& shape : shapes) {
            shape->draw();  // Polymorphic call (OVERRIDING)
        }
    }
};

int main() {

    Renderer3D renderer3D;
    
    GraphicsContext ctx(&renderer2D);
    
    // OVERLOADING: Same method name, different params
    ctx.add(100, 100, 50);        // Circle
    ctx.add(0, 0, 200, 200);      // Line
    
    cout << "=== 2D Rendering ===" << endl;
    ctx.drawAll();  // OVERRIDING: Each shape draws differently
    
    cout << "\n=== Switching to 3D Rendering ===" << endl;
    ctx.setRenderer(&renderer3D);
    ctx.drawAll();
    
    return 0;
}
```

## Mixin Classes in C++
Mixins are a powerful pattern for composing functionality by adding reusable behaviors to classes through inheritance or templates.

**What are Mixins?**
**Definition:** A mixin is a class that provides methods to other classes but isn't meant to be instantiated on its own. It's a way to "mix in" additional functionality.

``` cpp
┌─────────────────────────────────────────────────────────────────┐
│                    MIXIN PATTERN                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Traditional Inheritance:        Mixin Approach:                │
│                                                                 │
│      Base                        Class + Mixin1 + Mixin2        │
│       │                                  ↓                      │
│    Derived                          Composite Class            │
│                                                                 │
│  • Linear hierarchy                • Horizontal composition    │
│  • "Is-a" relationship             • "Has-behavior" relation   │
│  • Tight coupling                  • Flexible, reusable        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1. Basic Mixins (Simple Inheritance)
**Simple Example**
``` cpp
#include <iostream>
#include <string>
using namespace std;

// Mixin 1: Adds serialization capability
template<typename Derived>
class Serializable {
public:
    string serialize() const {
        return static_cast<const Derived*>(this)->serializeImpl();
    }
};

// Mixin 2: Adds logging capability
template<typename Derived>
class Loggable {
public:
    void log(const string& message) const {
        cout << "[" << typeid(Derived).name() << "] " << message << endl;
    }
};

// Mixin 3: Adds comparison capability
template<typename Derived>
class Comparable {
public:
    bool operator<(const Derived& other) const {
        return static_cast<const Derived*>(this)->compare(other) < 0;
    }
    
    bool operator>(const Derived& other) const {
        return static_cast<const Derived*>(this)->compare(other) > 0;
    }
    
    bool operator==(const Derived& other) const {
        return static_cast<const Derived*>(this)->compare(other) == 0;
    }
};

// User class that "mixes in" behaviors
class Person : public Serializable<Person>,
               public Loggable<Person>,
               public Comparable<Person> {
private:
    string name;
    int age;

public:
    Person(string n, int a) : name(n), age(a) {}
    
    // Required by Serializable
    string serializeImpl() const {
        return "Person{name=" + name + ", age=" + to_string(age) + "}";
    }
    
    // Required by Comparable
    int compare(const Person& other) const {
        if (age != other.age) return age - other.age;
        return name.compare(other.name);
    }
    
    string getName() const { return name; }
    int getAge() const { return age; }
};

int main() {
    Person alice("Alice", 30);
    Person bob("Bob", 25);
    
    // Using Serializable mixin
    cout << alice.serialize() << endl;
    
    // Using Loggable mixin
    alice.log("Person created");
    
    // Using Comparable mixin
    if (alice > bob) {
        cout << alice.getName() << " is older than " << bob.getName() << endl;
    }
    
    return 0;
}
```

**Output:**

```text

Person{name=Alice, age=30}
[6Person] Person created
Alice is older than Bob
```

### 2. CRTP (Curiously Recurring Template Pattern)
CRTP is the foundation of most mixin implementations in C++. The derived class passes itself as a template parameter to the base.

**Counter Mixin Example:**

``` cpp
    Countable(const Countable&) { ++count; }
    ~Countable() { --count; }

public:
```

 ### 4. Policy-Based Design with Mixins

 ``` cpp
 #include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

// Storage policies (mixins)
template<typename T>
class VectorStorage {
protected:
    vector<T> data_;
    
public:
    void add(const T& item) { data_.push_back(item); }
    size_t size() const { return data_.size(); }
    const T& get(size_t index) const { return data_[index]; }

    size_t size_ = 0;
    
public:

        if (size_ < N) data_[size_++] = item;
    }
    
    size_t size() const { return size_; }
    const T& get(size_t index) const { return data_[index]; }
    
    void clear() { size_ = 0; }
};

template<typename Derived>
class QuickSortable {
public:

        std::sort(self.begin(), self.end());

                    self.swap(j, j + 1);
                }
            }
        }

         template<typename> class SortPolicy>

    using StoragePolicy<T>::size_;

    auto end() { return data_.begin() + this->size(); }

        T temp = data_[i];
        data_[i] = data_[j];
        data_[j] = temp;
    }
    
    void display() const {
        cout << "Container: [";
        for (size_t i = 0; i < this->size(); i++) {
            cout << this->get(i);
            if (i < this->size() - 1) cout << ", ";
        }

    }
};
```

### 5. Observable Mixin Pattern

``` cpp
#include <iostream>
#include <vector>
#include <functional>
#include <string>
using namespace std;

// Observer interface
template<typename T>
class IObserver {
public:
    virtual void onNotify(const T& data) = 0;
    virtual ~IObserver() = default;
};

// Observable mixin
template<typename Derived, typename EventData>
class Observable {
private:
    vector<IObserver<EventData>*> observers_;

protected:
    void notify(const EventData& data) {
        for (auto* observer : observers_) {
            observer->onNotify(data);
        }
    }

public:
    void addObserver(IObserver<EventData>* observer) {
        observers_.push_back(observer);
    }
    
    void removeObserver(IObserver<EventData>* observer) {
        observers_.erase(
            remove(observers_.begin(), observers_.end(), observer),
            observers_.end()
        );
    }
};

// Event data structure
struct TemperatureEvent {
    double temperature;
    string unit;
};

// Concrete observable class
class TemperatureSensor : public Observable<TemperatureSensor, TemperatureEvent> {
private:
    double currentTemp_ = 0.0;

public:
    void setTemperature(double temp) {
        currentTemp_ = temp;
        notify({currentTemp_, "Celsius"});
    }
    
    double getTemperature() const { return currentTemp_; }
};

// Concrete observers
class Display : public IObserver<TemperatureEvent> {
private:
    string name_;

public:
    Display(string name) : name_(name) {}
    
    void onNotify(const TemperatureEvent& event) override {
        cout << name_ << " displays: " 
             << event.temperature << " " << event.unit << endl;
    }
};

class Alarm : public IObserver<TemperatureEvent> {
private:
    double threshold_;

public:
    Alarm(double threshold) : threshold_(threshold) {}
    
    void onNotify(const TemperatureEvent& event) override {
        if (event.temperature > threshold_) {
            cout << "⚠️  ALARM: Temperature exceeds " 
                 << threshold_ << " " << event.unit << "!" << endl;
        }
    }
};

class Logger : public IObserver<TemperatureEvent> {
public:
    void onNotify(const TemperatureEvent& event) override {
        cout << "[LOG] Temperature changed to: " 
             << event.temperature << " " << event.unit << endl;
    }
};

int main() {
    TemperatureSensor sensor;
    
    Display display1("Living Room Display");
    Display display2("Kitchen Display");
    Alarm highTempAlarm(30.0);
    Logger logger;
    
    // Register observers
    sensor.addObserver(&display1);
    sensor.addObserver(&display2);
    sensor.addObserver(&highTempAlarm);
    sensor.addObserver(&logger);
```

### 6. Mixin Stacking (Layered Mixins)

``` cpp
#include <iostream>
#include <string>
#include <chrono>
#include <thread>
using namespace std;

// Base class
class BasicWorker {
public:
    virtual void doWork() {
        cout << "Doing basic work" << endl;
    }
    virtual ~BasicWorker() = default;
};

// Mixin 1: Add timing
template<typename Base>
class TimedWorker : public Base {
public:
    void doWork() override {
        auto start = chrono::high_resolution_clock::now();
        
        cout << "⏱️  Starting timed work..." << endl;
        Base::doWork();
        
        auto end = chrono::high_resolution_clock::now();
        auto duration = chrono::duration_cast<chrono::milliseconds>(end - start);
        cout << "⏱️  Work completed in " << duration.count() << "ms" << endl;
    }
};

// Mixin 2: Add logging
template<typename Base>
class LoggedWorker : public Base {
private:
    int callCount_ = 0;

public:
    void doWork() override {
        ++callCount_;
        cout << "📝 [LOG " << callCount_ << "] Starting work..." << endl;
        Base::doWork();
        cout << "📝 [LOG " << callCount_ << "] Work finished" << endl;
    }
};

// Mixin 3: Add retry logic
template<typename Base>
class RetryableWorker : public Base {
private:
    int maxRetries_ = 3;

public:
    void setMaxRetries(int retries) { maxRetries_ = retries; }
    
    void doWork() override {
        for (int attempt = 1; attempt <= maxRetries_; ++attempt) {
            try {
                cout << "🔄 Attempt " << attempt << "/" << maxRetries_ << endl;
                Base::doWork();
                cout << "✅ Success!" << endl;
                return;
            } catch (const exception& e) {
                cout << "❌ Failed: " << e.what() << endl;
                if (attempt == maxRetries_) {
                    cout << "❌ All retries exhausted" << endl;
                    throw;
                }
            }
        }
    }
};

// Complex worker: Stack multiple mixins
using ComplexWorker = RetryableWorker<TimedWorker<LoggedWorker<BasicWorker>>>;

int main() {
    cout << "=== Simple Worker ===" << endl;
    BasicWorker simple;
    simple.doWork();
    
    cout << "\n=== Timed Worker ===" << endl;
    TimedWorker<BasicWorker> timed;
    timed.doWork();
    
    cout << "\n=== Logged + Timed Worker ===" << endl;
    TimedWorker<LoggedWorker<BasicWorker>> loggedTimed;
    loggedTimed.doWork();
    
    cout << "\n=== Complex Worker (All Mixins) ===" << endl;
    ComplexWorker complex;
    complex.setMaxRetries(2);
    complex.doWork();
    
    return 0;
}
```

**Output:**

```text

Doing basic work

=== Timed Worker ===
⏱️  Starting timed work...
Doing basic work
⏱️  Work completed in 0ms

=== Logged + Timed Worker ===
⏱️  Starting timed work...
📝 [LOG 1] Starting work...
Doing basic work
📝 [LOG 1] Work finished
⏱️  Work completed in 0ms
```

### 7. Real-World Example: Game Entity System
``` cpp
#include <iostream>
#include <string>
#include <cmath>
using namespace std;

// Position mixin
template<typename Derived>
class Positionable {
protected:
    double x_ = 0, y_ = 0;

public:
    Derived& setPosition(double x, double y) {
        x_ = x;
        y_ = y;
        return static_cast<Derived&>(*this);
    }
    
    double getX() const { return x_; }
    double getY() const { return y_; }
    
    double distanceTo(double x, double y) const {
        return sqrt((x_ - x) * (x_ - x) + (y_ - y) * (y_ - y));
    }
};

// Movable mixin
template<typename Derived>
class Movable : public Positionable<Derived> {
protected:
    double velocityX_ = 0, velocityY_ = 0;

public:
    Derived& setVelocity(double vx, double vy) {
        velocityX_ = vx;
        velocityY_ = vy;
        return static_cast<Derived&>(*this);
    }
    
    void update(double deltaTime) {
        this->x_ += velocityX_ * deltaTime;
        this->y_ += velocityY_ * deltaTime;
    }
};

// Renderable mixin
template<typename Derived>
class Renderable {
protected:
    string sprite_;

public:
    Derived& setSprite(const string& sprite) {
        sprite_ = sprite;
        return static_cast<Derived&>(*this);
    }
    
    void render() const {
        cout << "Rendering " << sprite_ << " at ("
             << static_cast<const Derived*>(this)->getX() << ", "
             << static_cast<const Derived*>(this)->getY() << ")" << endl;
    }
};

// Health mixin
template<typename Derived>
class Damageable {
protected:
    int health_ = 100;
    int maxHealth_ = 100;

public:
    Derived& setHealth(int health, int maxHealth) {
        health_ = health;
        maxHealth_ = maxHealth;
        return static_cast<Derived&>(*this);
    }
    
    void takeDamage(int damage) {
        health_ -= damage;
        if (health_ < 0) health_ = 0;
        
        cout << static_cast<Derived*>(this)->getName() 
             << " takes " << damage << " damage! "
             << "Health: " << health_ << "/" << maxHealth_ << endl;
             
        if (health_ == 0) {
            cout << "💀 " << static_cast<Derived*>(this)->getName() 
                 << " has been destroyed!" << endl;
        }
    }
    
    bool isAlive() const { return health_ > 0; }
    int getHealth() const { return health_; }
};

// Named mixin
template<typename Derived>
class Named {
protected:
    string name_;

public:
    Derived& setName(const string& name) {
        name_ = name;
        return static_cast<Derived&>(*this);
    }
    
    const string& getName() const { return name_; }
};

// Game entities using different mixin combinations

// Player: Can move, take damage, has position, name, and sprite
class Player : public Movable<Player>,
               public Damageable<Player>,
               public Renderable<Player>,
               public Named<Player> {
public:
    Player() {
        setName("Player")
            .setSprite("player.png")
            .setHealth(100, 100)
            .setPosition(0, 0);
    }
    
    void displayStatus() const {
        cout << "🎮 " << name_ << " - Health: " << health_ 
             << "/" << maxHealth_ << " - Position: (" 
             << x_ << ", " << y_ << ")" << endl;
    }
};

// Enemy: Can move, take damage, has position and name
class Enemy : public Movable<Enemy>,
              public Damageable<Enemy>,
              public Named<Enemy> {
public:
    Enemy(const string& name) {
        setName(name)
            .setHealth(50, 50)
            .setPosition(100, 100);
    }
    
    void displayStatus() const {
        cout << "👾 " << name_ << " - Health: " << health_ 
             << "/" << maxHealth_ << " - Position: (" 
             << x_ << ", " << y_ << ")" << endl;
    }
};

// StaticObject: Only has position and name (can't move or take damage)
class StaticObject : public Positionable<StaticObject>,
                     public Named<StaticObject> {
public:
    StaticObject(const string& name, double x, double y) {
        setName(name).setPosition(x, y);
    }
    
    void displayInfo() const {
        cout << "🏛️  " << name_ << " at (" << x_ << ", " << y_ << ")" << endl;
    }
};

int main() {
    Player player;
    player.setVelocity(10, 5);
    
    Enemy goblin("Goblin");
    goblin.setVelocity(-5, -3);
    
    StaticObject tree("Ancient Tree", 50, 75);
    
    cout << "=== Initial State ===" << endl;

    goblin.update(1.0);
    player.displayStatus();
    goblin.displayStatus();

        cout << "Player counterattacks!" << endl;
        goblin.takeDamage(30);
    }
    
    cout << "\n=== Rendering ===" << endl;
    player.render();
    
    return 0;
}
```

### 9. Best Practices

``` cpp
// ✅ DO: Use CRTP for type-safe mixins
template<typename Derived>
class Mixin {
    Derived& self() { return static_cast<Derived&>(*this); }
};

// ✅ DO: Document required interface
template<typename Derived>
class Sortable {
    // Requires: Derived must have size(), swap(), and compare()
};

// ✅ DO: Use concepts (C++20) to enforce requirements
template<typename T>
concept HasSize = requires(T t) {
    { t.size() } -> std::convertible_to<size_t>;
};

template<typename Derived>
requires HasSize<Derived>
class Container { };

// ❌ DON'T: Create too deep inheritance hierarchies
// Bad: Too many layers
using TooComplex = Mixin5<Mixin4<Mixin3<Mixin2<Mixin1<Base>>>>>;

// ✅ DO: Keep it shallow and clear
using Clear = Loggable<Serializable<Base>>;

// ✅ DO: Provide clear names
template<typename Derived>
class ThreadSafeMixin { };  // Clear purpose

// ❌ DON'T: Use generic names
template<typename Derived>
class HelperMixin { };  // What does it help with?
```

### Summary
**Mixins in C++:**
Basic Mixins: Simple template-based inheritance
CRTP: Pass derived class as template parameter
Chainable: Enable fluent interfaces
Policy-Based: Mix and match behaviors
Stacking: Layer multiple mixins

**When to use:**
✅ Cross-cutting concerns (logging, serialization)
✅ Optional features (undo/redo, caching)
✅ Reusable behaviors across unrelated classes
✅ Compile-time polymorphism

**When NOT to use:**
❌ Simple inheritance would suffice
❌ Runtime polymorphism is needed
❌ Behavior depends on runtime data
Mixins are a powerful tool for creating flexible, reusable, and type-safe components! 🚀
