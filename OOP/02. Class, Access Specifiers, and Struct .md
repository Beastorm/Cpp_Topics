# Classes, Objects, Access Specifiers & Structs in C++

---

## 1. Classes and Objects

> **Class** = Blueprint | **Object** = Instance created from blueprint
```
  Car Blueprint (Class)          Specific Cars (Objects)
  ┌─────────────────┐            car1 → "Red Toyota 2022"
  │ brand: string   │   ──→      car2 → "Blue Tesla 2024"
  │ year: int       │            car3 → "White BMW 2023"
  │ drive()         │
  └─────────────────┘
```

### Basic Syntax
```cpp
class ClassName {
public:
    int variable;        // Data Member

    void function() {   // Member Function
        // logic
    }
};  // ← semicolon is MANDATORY
```

### Simple Example
```cpp
#include <iostream>
#include <string>
using namespace std;

class Student {
public:
    string name;
    int rollNumber;

    void introduce() {
        cout << "Hi, I am " << name << " (ID: " << rollNumber << ")\n";
    }
};

int main() {
    Student s1;             // Create object
    s1.name = "Alice";      // Access via dot operator
    s1.rollNumber = 101;
    s1.introduce();         // Call method

    Student s2;
    s2.name = "Bob";
    s2.rollNumber = 102;
    s2.introduce();
}
```
```
Output:
Hi, I am Alice (ID: 101)
Hi, I am Bob (ID: 102)
```

### Professional Example (with Constructor + Encapsulation)
```cpp
class Car {
private:
    string brand;   // hidden data
    int year;

public:
    Car(string b, int y) : brand(b), year(y) {}  // Constructor

    void displayInfo() {
        cout << year << " " << brand << "\n";
    }
};

int main() {
    Car car1("Toyota", 2022);  // initialized immediately via constructor
    Car car2("Tesla", 2024);
    car1.displayInfo();  // 2022 Toyota
    car2.displayInfo();  // 2024 Tesla
}
```

### Key Terms

| Term | Definition |
|---|---|
| **Class** | Blueprint/template defining structure |
| **Object** | Actual variable created from class |
| **Member** | Variables and functions inside the class |
| **Constructor** | Special function that runs when object is created |

## 2. Access Specifiers

> **Control WHO can access your data.**

### The House Analogy
```
  ┌──────────────────────────────────────────────┐
  │                  HOUSE                       │
  │                                              │
  │  🚪 Front Porch  →  PUBLIC                  │
  │     (anyone can see/use)                    │
  │                                              │
  │  📺 Family TV    →  PROTECTED               │
  │     (family + children only)                │
  │                                              │
  │  📔 Personal Diary → PRIVATE                │
  │     (only you — no one else!)               │
  └──────────────────────────────────────────────┘
```

### Who Can Access What?
```
  ┌────────────────┬───────────┬──────────────┬──────────────┐
  │                │  public   │  protected   │   private    │
  ├────────────────┼───────────┼──────────────┼──────────────┤
  │ Same class     │    ✓      │      ✓       │      ✓       │
  │ Derived class  │    ✓      │      ✓       │      ✗       │
  │ main() / user  │    ✓      │      ✗       │      ✗       │
  └────────────────┴───────────┴──────────────┴──────────────┘
```

### Complete Example
```cpp
#include <iostream>
using namespace std;

class Base {
public:
    int x = 10;      // everyone can access

protected:
    int y = 20;      // only Base and its children

private:
    int z = 30;      // only Base itself

public:
    void showAll() {
        cout << x << " " << y << " " << z << "\n";  // ✓ all OK inside class
    }
};

class Derived : public Base {
public:
    void show() {
        cout << x << "\n";  // ✓ public
        cout << y << "\n";  // ✓ protected (inherited)
     // cout << z;          // ✗ ERROR — private in Base
    }
};

int main() {
    Base b;
    cout << b.x;    // ✓ public
 // cout << b.y;   // ✗ protected — not from main
 // cout << b.z;   // ✗ private  — not from main
    b.showAll();    // ✓ ask class to reveal its own data

    Derived d;
    d.show();       // ✓ derived class handles protected access
}
```

---

### public
```cpp
class Sample {
public:
    int num;
};

Sample obj;
obj.num = 5;   // ✓ valid anywhere
```

**Use for:** Functions meant for the user (`printInfo()`, `login()`, `getBalance()`)

---

### private (default for class)
```cpp
class Sample {
private:
    int secret;          // hidden

public:
    void setSecret(int s) { secret = s; }    // setter
    int getSecret()       { return secret; } // getter
};

Sample obj;
// obj.secret = 10;      // ✗ ERROR
obj.setSecret(10);       // ✓ use public method instead
```

**Use for:** Sensitive data (passwords, balances, internal state)

---

### protected
```cpp
class BankAccount {
protected:
    double balance;      // children can access, main cannot

public:
    void setBalance(double b) { balance = b; }
};

class SavingsAccount : public BankAccount {
public:
    void addInterest() {
        double interest = balance * 0.05;  // ✓ direct access (child class)
        balance += interest;
        cout << "New Balance: " << balance << "\n";
    }
};

int main() {
    SavingsAccount acc;
    acc.setBalance(1000);    // ✓ public method
    acc.addInterest();       // ✓ child handles protected data
 // acc.balance = 5000;      // ✗ ERROR — protected
}
```

**Use for:** Data shared between parent and child classes

---

### protected — Inheritance Mode Rules
```
  class Child : [mode] Parent

  ┌──────────────┬──────────────────────────────────────────────┐
  │ Mode         │ How parent's members appear in Child         │
  ├──────────────┼──────────────────────────────────────────────┤
  │ public       │ public→public, protected→protected           │
  │ protected    │ public→protected, protected→protected        │
  │ private      │ public→private, protected→private            │
  │              │ (grandchildren can't see — chain breaks)     │
  └──────────────┴──────────────────────────────────────────────┘
```

### protected — Quick Checklist
```
  Can main() access it?          ✗ No
  Can the class itself?          ✓ Yes
  Can a child class?             ✓ Yes
  Can a grandchild (public inh)? ✓ Yes
```

## 3. Struct

> **Group related variables of different types under one name.**
```cpp
struct Student {
    string name;
    int age;
    float gpa;
};  // ← semicolon required

// Create and use
Student s1;
s1.name = "John";
s1.age = 20;
s1.gpa = 3.8;

// Brace initialization (shortcut)
Student s2 = {"Jane", 22, 3.9};
```

### Struct with Constructor and Functions
```cpp
struct Point {
    int x, y;

    Point(int xVal, int yVal) : x(xVal), y(yVal) {}  // constructor

    void display() {
        cout << "(" << x << ", " << y << ")\n";
    }
};

Point p1(10, 20);
p1.display();   // (10, 20)
```

> In C++, structs can have constructors and functions — just like classes!

## 4. Struct vs Class

### The Only Technical Differences
```
  ┌──────────────────────┬──────────────┬──────────────┐
  │ Feature              │   class      │   struct     │
  ├──────────────────────┼──────────────┼──────────────┤
  │ Default member access│  private     │  public      │
  │ Default inheritance  │  private     │  public      │
  │ Can have functions?  │  ✓ Yes       │  ✓ Yes       │
  │ Can have constructor?│  ✓ Yes       │  ✓ Yes       │
  └──────────────────────┴──────────────┴──────────────┘
```

### Proof of Default Access Difference
```cpp
class MyClass {
    int x;   // private by default
};

struct MyStruct {
    int x;   // public by default
};

int main() {
    MyClass c;
 // c.x = 10;  // ✗ ERROR — private

    MyStruct s;
    s.x = 10;   // ✓ OK — public
}
```

### Proof of Default Inheritance Difference
```cpp
class Base { public: int x; };

class DerivedClass  : Base {};    // private inheritance by default
struct DerivedStruct : Base {};   // public  inheritance by default

int main() {
    DerivedClass  d1;
 // d1.x = 10;   // ✗ ERROR — x became private

    DerivedStruct d2;
    d2.x = 10;   // ✓ OK — x stays public
}
```

### When to Use Which?
```
  Use struct when:                   Use class when:
  ┌─────────────────────────┐       ┌─────────────────────────────┐
  │ Simple data container   │       │ Complex object with logic    │
  │ No logic needed         │       │ Encapsulation required       │
  │ All data is public      │       │ Private data + getters       │
  │ Examples:               │       │ Examples:                    │
  │   Point {x, y}          │       │   BankAccount                │
  │   Color {r, g, b}       │       │   Student (with validation)  │
  │   Config {host, port}   │       │   Car (with methods)         │
  └─────────────────────────┘       └─────────────────────────────┘

  Convention:
  struct → "This is open, simple data"   (other devs see it immediately)
  class  → "This has logic and privacy"  (careful, encapsulated design)
```

## Quick Reference Summary
```
  ┌────────────────────────────────────────────────────────────┐
  │  Access Specifier   │  Same Class  │  Child  │  main()    │
  ├─────────────────────┼──────────────┼─────────┼────────────┤
  │  public             │     ✓        │    ✓    │    ✓       │
  │  protected          │     ✓        │    ✓    │    ✗       │
  │  private (default)  │     ✓        │    ✗    │    ✗       │
  └─────────────────────┴──────────────┴─────────┴────────────┘

  class  default = private  →  good for OOP with encapsulation
  struct default = public   →  good for simple data bundles
```
