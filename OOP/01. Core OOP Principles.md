# Core OOP Principles in C++

---

## 1. Encapsulation

> **Bundling data + functions into one unit (class), and hiding internal details from outside.**

### The Problem Without Encapsulation
```cpp
struct BankAccount {
    double balance;  // public — anyone can set balance = -99999 !
};
```

### The Solution With Encapsulation
```cpp
class BankAccount {
private:
    double balance;        // hidden — no direct access

public:
    BankAccount(double initial) : balance(initial) {}

    void deposit(double amount) {
        if (amount > 0) balance += amount;
    }

    bool withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            return true;
        }
        return false;
    }

    double getBalance() const { return balance; }
};

// Usage:
BankAccount acc(100.0);
acc.deposit(50.0);          // ✓ OK
acc.withdraw(30.0);         // ✓ OK
// acc.balance = -9999;     // ✗ ERROR — balance is private
```

### Key Benefits
```
  ┌─────────────────────────────────────────────────────┐
  │               CLASS (Black Box)                     │
  │                                                     │
  │   [private data]  ←→  [public methods]  ←── user   │
  │    balance             deposit()                    │
  │    data[]              withdraw()                   │
  │                        getBalance()                 │
  └─────────────────────────────────────────────────────┘

  ✓ Protects internal state from invalid use
  ✓ Enforces business rules (no negative deposits)
  ✓ Implementation can change without breaking user code
```

## 2. Abstraction

> **Showing only essential features, hiding complex internal details.**

| | What it hides | How |
|---|---|---|
| **Encapsulation** | Data | `private` members |
| **Abstraction** | Complexity | Simple public interface |

### Example: Shape Interface
```cpp
// Abstract class — defines WHAT, not HOW
class Shape {
public:
    virtual double area() const = 0;  // pure virtual
    virtual ~Shape() = default;
};

class Circle : public Shape {
    double radius;
public:
    Circle(double r) : radius(r) {}
    double area() const override { return 3.14159 * radius * radius; }
};

class Rectangle : public Shape {
    double width, height;
public:
    Rectangle(double w, double h) : width(w), height(h) {}
    double area() const override { return width * height; }
};

// Usage — caller doesn't know HOW area is computed
Shape* s1 = new Circle(5.0);
Shape* s2 = new Rectangle(4.0, 6.0);
cout << s1->area();  // 78.53
cout << s2->area();  // 24.0
```
```
  User sees:          User doesn't see:
  ┌────────────┐      ┌──────────────────────────┐
  │ area()     │      │ Circle:  π × r²           │
  │            │  ←   │ Rect:    w × h             │
  │ (simple)   │      │ Triangle: ½ × base × h     │
  └────────────┘      └──────────────────────────┘
```

## 3. Inheritance

> **A class (child) acquires properties and behaviors of another class (parent).**

### Basic Syntax
```cpp
class Parent {
    // parent code
};

class Child : public Parent {
    // inherits everything from Parent
    // can add its own members too
};
```

### IS-A Relationship
```
  Animal ──── Dog       (Dog IS-A Animal ✓)
  Vehicle ─── Car       (Car IS-A Vehicle ✓)
  Shape ────┬ Circle    (Circle IS-A Shape ✓)
            └ Rectangle
```

---

### Type A: Single Inheritance
```cpp
class Animal {
public:
    void eat()   { cout << "I can eat!\n"; }
    void sleep() { cout << "I can sleep!\n"; }
};

class Dog : public Animal {
public:
    void bark() { cout << "Woof woof!\n"; }
};

Dog myDog;
myDog.eat();    // from Animal
myDog.sleep();  // from Animal
myDog.bark();   // from Dog
```

---

### Type B: Multilevel Inheritance
```
  Fruit ──→ Citrus ──→ Orange
  (grandparent) (parent) (child)
```
```cpp
class Fruit  { public: void origin() { cout << "From plants\n"; } };
class Citrus : public Fruit  { public: void type() { cout << "Citrus\n"; } };
class Orange : public Citrus { public: void color(){ cout << "Orange\n"; } };

Orange o;
o.origin();  // from Fruit
o.type();    // from Citrus
o.color();   // from Orange
```

---

### Type C: Multiple Inheritance
```cpp
class Engine { public: void start() { cout << "Engine on\n"; } };
class Wheels { public: void roll()  { cout << "Rolling\n"; } };

class Car : public Engine, public Wheels {
public:
    void drive() { cout << "Moving!\n"; }
};

Car c;
c.start();  // from Engine
c.roll();   // from Wheels
c.drive();  // from Car
```

---

### Access Specifiers
```
  When you write:  class Child : [mode] Parent

  ┌──────────────┬──────────────┬────────────────┬───────────────┐
  │ Parent member│ public mode  │ protected mode │ private mode  │
  ├──────────────┼──────────────┼────────────────┼───────────────┤
  │ public       │ public       │ protected      │ private       │
  │ protected    │ protected    │ protected      │ private       │
  │ private      │ inaccessible │ inaccessible   │ inaccessible  │
  └──────────────┴──────────────┴────────────────┴───────────────┘

  public    → everyone (main, child class)
  protected → only child classes (not main)
  private   → only the class itself
```
```cpp
class Vehicle {
protected:
    int speed;         // child can access, main cannot
public:
    void setSpeed(int s) { speed = s; }
};

class Car : public Vehicle {
public:
    void display() { cout << speed << " km/h\n"; }  // ✓ works
};

Car c;
c.setSpeed(100);    // ✓ public method
c.display();        // ✓ ok
// c.speed = 200;   // ✗ ERROR — protected
```

## 4. Polymorphism

> **"Many forms" — one function/operator behaves differently based on context.**
```
  Two Types:
  ┌─────────────────────────────────────────────────┐
  │ 1. Compile-time (Static)  → Function Overloading │
  │ 2. Run-time    (Dynamic)  → Function Overriding  │
  └─────────────────────────────────────────────────┘
```

---

### Type 1: Compile-time Polymorphism (Function Overloading)

> **Same function name, different parameters → compiler chooses at compile time.**
```cpp
class Calculator {
public:
    void add(int a, int b)            { cout << a + b << "\n"; }
    void add(double a, double b)      { cout << a + b << "\n"; }
    void add(int a, int b, int c)     { cout << a + b + c << "\n"; }
};

Calculator calc;
calc.add(10, 20);       // calls add(int, int)
calc.add(5.5, 2.3);     // calls add(double, double)
calc.add(1, 2, 3);      // calls add(int, int, int)
```

---

### Type 2: Run-time Polymorphism (Function Overriding)

> **`virtual` functions — correct version chosen at RUNTIME based on actual object type.**
```cpp
class Animal {
public:
    virtual void speak() {        // 'virtual' = decide at runtime
        cout << "Generic sound\n";
    }
};

class Dog : public Animal {
public:
    void speak() override {       // 'override' = confirm we're overriding
        cout << "Woof!\n";
    }
};

class Cat : public Animal {
public:
    void speak() override {
        cout << "Meow!\n";
    }
};

Animal* ptr;
Dog d; Cat c;

ptr = &d; ptr->speak();   // Woof!   ← calls Dog's version
ptr = &c; ptr->speak();   // Meow!   ← calls Cat's version
```
```
  WITHOUT virtual:         WITH virtual:
  ptr = &dog;              ptr = &dog;
  ptr->speak();            ptr->speak();
       ↓                        ↓
  "Generic sound"          "Woof!"       ← correct!
  (always base class)      (actual object type)
```

---

### Abstract Classes (Pure Virtual Functions)

> **Force derived classes to implement a function. Cannot instantiate abstract class.**
```cpp
class Shape {
public:
    virtual void draw() = 0;   // "= 0" → pure virtual → abstract class
    virtual ~Shape() = default;
};

class Circle : public Shape {
public:
    void draw() override { cout << "Drawing Circle\n"; }
};

class Rectangle : public Shape {
public:
    void draw() override { cout << "Drawing Rectangle\n"; }
};

// Shape s;    // ✗ ERROR — can't instantiate abstract class
Shape* s1 = new Circle();     // ✓
Shape* s2 = new Rectangle();  // ✓
s1->draw();   // Drawing Circle
s2->draw();   // Drawing Rectangle
```

## Quick Reference Summary
```
  ┌──────────────────┬───────────────────────────────────────────────┐
  │ Principle        │ Core Idea                                     │
  ├──────────────────┼───────────────────────────────────────────────┤
  │ Encapsulation    │ Hide data (private), expose via methods       │
  │ Abstraction      │ Hide complexity, show simple interface        │
  │ Inheritance      │ Child reuses parent's code (IS-A relation)    │
  │ Polymorphism     │ One interface, many forms/behaviors           │
  └──────────────────┴───────────────────────────────────────────────┘

  Polymorphism types:
  ┌──────────────────┬──────────────────┬──────────────────────────┐
  │ Type             │ Mechanism        │ Decided When?            │
  ├──────────────────┼──────────────────┼──────────────────────────┤
  │ Compile-time     │ Overloading      │ Compile time (faster)    │
  │ Run-time         │ virtual + override│ Runtime (flexible)      │
  └──────────────────┴──────────────────┴──────────────────────────┘

  Inheritance access:
  public    → IS-A relationship, full public interface preserved
  protected → implementation reuse, restricted outside access
  private   → strictest reuse, no polymorphism
```
